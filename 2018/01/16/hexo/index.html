Hexo 快速建站

2018-01-16

使用 hexo + GitPage，来搭建静态博客那是相当 easy。用 Markdown 写文章做记录，再push 到 Git 上 ，简单快捷。

首先保证你电脑里的 Node.js Git 已经安装。
使用 npm 完成 Hexo 的安装。

具体请参考: 这里

生成的目录结构

找到全局配置文件的 _config.yml 修改网站基本信息

基本操作

页面信息

如果不想博文在首页全部显示, 并能出现阅读全文按钮效果, 需要在你想在首页显示的部分下添加

更换主题

选一个你喜欢的主题
我以hueman为例,下载之后更改文件夹名为hueman，放在 themes 文件夹内，修改全局配置文件

为例支持内置的 search 需要下载 hexo-generator-json-content

详细见文档

主题目录下的 _config.yml 为主题配置信息,可以修改你所需要的配置

文章多级分类

想在侧边栏显示多级的文章分类，记得在 _config.yml 中开启 Categories: 。
在文章.md头部加上

如果想支持更多级分类，去 hueman/layout/widget/category.ejs

文章缩略图

想试用文档缩略图，记得在 _config.yml 中开启 thumbnail: true 。
默认会选取文章的第一张图作为缩略图，你也可以自定义啦，去文章.md在头部加上

文件路径有个坑，文档里面使用了一个外链没有任何问题，但是如果想试用站内链接，我们把图片放在hueman/source/css/images 内,在 page 的页面里面 sidebar 的预览图就会因为路径错误显示不正确。
于是我参考了一下 页面 logo 的处理方式，打开 hueman/layout/common/thumbnail.ejs

这样就可以把图片保存在本地啦～

在文章中插入图片

提供两种插入图片的方法

  • 我们可以在全局配置文件 _config.yml

这样我们每次new,一个文章的时候,就会同时生成一个同名的文件夹，可以在里面放入图片，随后在文章中引用

  • 因为自己在文章中插入的图片不多，所以直接在 source 里面新建了一个文件夹 img ，把需要引用的图片放在了img/内，随后在文章中以绝对路径引用
