JavaScript 中 this 的理解

2018-01-31

this到底指向哪里

在JavaScript 中，函数中this到指向，经常会引起自己的困惑，所以写一篇文章来梳理下自己对这两个疑点的理解。

执行时指定

一般而言JavaScript中，this 指向函数执行的当前对象

举个栗子

this 指向了 showName 函数运行时的对象b

emmm… 那没有明确的执行对象呢，我们来看下一个栗子

b 函数执行时没有明确的执行对象，此时函数中的 this 就会指向全局（浏览器中为 window node 为 global）。
浏览器中相当于执行的为 window.b()，this 指向window， window.name 输出 jack。

好，我相信到这里已经解释的差不多了，看下面这个栗子

是否能一眼看出答案呢，我们来看看函数里面 this 指向

匿名函数 和 setTimeout,setInterval函数执行时 this 的对象时全局对象

匿名函数 函数执行时 this 的对象时全局对象,setTimeout 和 setInterval执行时的函数内的this 也是

setTimeout中的延迟执行的代码中的this永远都指向window。
再看一个

对比一下

有个共同的现象如果setTimeout(字符串代码, 延迟) 使用这种方式执行的函数，默认会在window全局作用域下创建一个新的函数。此时this 会指向 window。

总结一下

  • setTimeout中的延迟执行的代码中的this永远都指向window
  • setTimeout(this.showName,1000) 这种形式中的this，即上文中提到的第一个this，是根据上下文来判断的
  • setTimeout(“this.showName()”,1000) 执行代码如果是字符串形式的代码，默认在window全局作用域下创建一个新的函数
